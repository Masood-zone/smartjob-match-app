# Smart Job Matching

Smart Job Matching is a Next.js App Router application for connecting system admins, employers, and job seekers in one hiring workflow. It uses Prisma, Better Auth, TanStack Query, PostgreSQL, Cloudinary, and React Hook Form to support onboarding, job posting, applications, matching, interviews, dashboards, and admin review.

## Overview

The application is structured around four layers:

1.  UI routes and screens in `app/`
2.  API route handlers in `app/api/`
3.  Client data hooks and request helpers in `services/`
4.  Database access and business rules in `lib/` and `prisma/`

Prisma is the source of truth for data access, and the main workflows are driven through onboarding, job creation, application tracking, and deterministic matching.

## How The System Works

### 1. System admin flow

The admin manages the platform, reviews onboarding submissions, controls verification status, and configures the matching algorithm.

Admin users can:

- View platform-wide dashboard metrics
- Review job seeker records and approve or reject them
- Review employer records and approve or reject them
- Inspect and update the matching algorithm configuration
- Access the admin setup flow for initial platform bootstrapping

Admin functionality is exposed through the admin area in `app/admin/` and the admin API routes in `app/api/admin/`.

### 2. Employer flow

Employers go through onboarding first. Once their profile is completed and verified, they can access the employer dashboard and begin hiring.

Employers can:

- Complete onboarding with company, team, and review data
- Upload branding or supporting files through the shared uploads endpoint
- View their dashboard and company summary
- Create job posts
- Review applicants for each job
- Inspect candidate matches
- Schedule interviews
- View analytics and settings for their company profile

Employer pages live under `app/onboarding/employer/` and the employer dashboard API is available at `/api/employer/dashboard`.

### 3. Job seeker flow

Job seekers complete a profile-driven onboarding flow that captures identity, qualification, experience, and review data. After onboarding, they can explore jobs, apply, and track outcomes from their dashboard.

Job seekers can:

- Complete onboarding and submit profile details
- Browse jobs and companies
- Apply for jobs
- Track application status and interview details
- View job matches generated by the matching engine
- Review analytics and dashboard summaries
- Update their settings from the dashboard

Job seeker pages live under `app/onboarding/job-seeker/` and the job seeker dashboard API is available at `/api/job-seeker/dashboard`.

### 4. Matching flow

Matching is deterministic and based on explicit profile and job data rather than AI scoring. The algorithm uses qualification, skills, experience, and preference alignment to rank candidates.

The matching pipeline is described in `docs/Matching-Algorithm.md` and is triggered by onboarding completion and related profile updates.

### 5. Job qualification matching algorithm

The job qualification matching algorithm is the scoring layer that links job seeker profiles to employer job posts across the system. It works as a pipeline rather than a single score so the platform can explain why a candidate matched, why they were filtered out, and how they compare to other seekers.

The system uses these data sources:

- Job seeker qualification, skills, experience, and location preference from onboarding data
- Job requirements such as required qualification, title, description, and required skills from job posts
- Application state and interview state for dashboard and review views
- Admin-tuned algorithm weights and verification rules from the admin configuration layer

The matching process follows this order:

1. Qualification gate: the system compares the job seeker qualification against the job requirement and rejects or deprioritizes underqualified profiles.
2. Skills scoring: the system measures overlap between seeker skills and job requirements and adds weighted points.
3. Experience scoring: the system checks years of experience, role relevance, and keyword similarity in past work history.
4. Preference scoring: the system checks whether the role fits the seeker’s stated location and preference data.
5. Final normalization: the score is capped and converted into a readable match band such as excellent, good, average, or low.

The output of the algorithm is stored as a match record and becomes visible in three places:

- Admin view: admins can review matched candidates while overseeing verification and algorithm settings.
- Employer view: employers see top matches, applicant ranking, and candidate detail cards in their dashboard and applicant screens.
- Job seeker view: seekers see matched jobs, application progress, and dashboard summaries of their best opportunities.

The algorithm is refreshed when any of these events happen:

- Job seeker onboarding is completed or updated
- A new job is created or edited
- An admin approves a job seeker profile
- The admin changes matching weights or rules

This keeps the platform consistent: the same qualification data that is collected during onboarding is reused by the job listing, matching, application, and dashboard layers.

## Role Scope

| Module                      | Admin can do                                 | Employer can do                               | Job seeker can do                                        |
| --------------------------- | -------------------------------------------- | --------------------------------------------- | -------------------------------------------------------- |
| Authentication              | Initialize platform setup and manage access  | Sign in and complete onboarding               | Sign in, register, and recover access                    |
| Onboarding                  | Review and verify submissions                | Submit company and team details               | Submit identity, qualification, and experience details   |
| Dashboard                   | View platform metrics and status             | View hiring metrics and company activity      | View applications, matches, analytics, and notifications |
| Job management              | Inspect platform jobs and employer activity  | Create and manage jobs                        | Browse jobs and apply                                    |
| Applicants and matches      | Review system-level records                  | Review applicants and matches for posted jobs | Track personal match results and application progress    |
| Verification                | Approve or reject employers and job seekers  | Await admin approval                          | Await admin approval                                     |
| Algorithm settings          | View and update scoring configuration        | No access                                     | No access                                                |
| Notifications and messaging | Trigger admin decisions and platform notices | Receive onboarding and hiring updates         | Receive application and interview updates                |

## Key User Journeys

### Admin to employer

1.  Admin reviews an employer onboarding submission.
2.  The employer is approved or rejected with a reason.
3.  Once approved, the employer can access their dashboard and start posting jobs.
4.  New jobs feed into the public listings, applicant tracking, and matching engine.

### Employer to job seeker

1.  Employer creates a job post with qualification and skill requirements.
2.  The system exposes the job through public listings and dashboard views.
3.  Matching identifies suitable job seekers and stores match records.
4.  Employers review applicants, inspect candidate details, and schedule interviews.

### Job seeker to employer

1.  Job seeker completes onboarding and is verified.
2.  The system matches the seeker to relevant jobs based on their profile.
3.  The seeker applies for a job and can track the application state.
4.  The employer can accept, reject, or move the application into interview flow.

## Service Layer

Business logic and client data hooks are grouped by domain in `services/`.

- `services/admin/` for admin dashboard, verification, and algorithm hooks
- `services/auth/` for auth-related client helpers
- `services/jobs.ts` for job listing, creation, and applicant queries
- `services/applications.ts` for application submission and status updates
- `services/matches.ts` for match retrieval
- `services/interviews.ts` for interview scheduling
- `services/companies.ts` for company discovery and detail queries
- `services/employer/dashboard.ts` for employer dashboard data
- `services/job-seeker/dashboard.ts` for job seeker dashboard data
- `services/onboarding/` for employer and job seeker onboarding persistence

React Query hooks live alongside the related service modules.
